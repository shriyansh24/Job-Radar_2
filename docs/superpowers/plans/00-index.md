# Scraper Platform Implementation Plan — Master Index

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Transform the scraping subsystem from a collection of scraper components into a policy-driven ingestion platform with tiered execution, ATS auto-detection, 1,473 H1B career page support, and scoring-based scheduling.

**Architecture:** Modular monolith with clean service boundaries. Six logical planes (control, execution, extraction, data quality, enrichment, observability) inside the FastAPI backend. New tables: `scrape_targets`, `scrape_attempts`. New ports: `FetcherPort`, `BrowserPort`, `ExtractorPort`. Registry-driven ATS classification. Scoring-based scheduler. CLI ops tool.

**Tech Stack:** Python 3.13, FastAPI, SQLAlchemy 2.0 (async), PostgreSQL, httpx, Scrapling, Crawl4AI, Nodriver, Cloudscraper, Camoufox, SeleniumBase, BrowserForge, fake-useragent, Protego, typer, rich, pytest + pytest-asyncio

**Spec:** `docs/superpowers/specs/2026-03-19-scraper-platform-design.md`
