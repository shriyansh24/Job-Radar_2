Do not use this file as the current source of truth for scheduler/process boundaries, compose services, workflow coverage, or local runtime commands.

Root scope: `D:/jobradar-v2/infra`, root operational files, Docker/Compose artifacts, and GitHub workflows.

## Infrastructure Summary

- Primary runtime model: Docker Compose
- Base services in `docker-compose.yml`: `backend`, `frontend`, `postgres`, `redis`
- Dev overlay: `docker-compose.dev.yml`
- CI: GitHub Actions only
- Deployment manifests: none found
- Worker deployment: not separate; scheduler/workers run inside backend API process

## Local Development Boot Flow

### Containerized path

1. `docker-compose.yml` starts backend, frontend, postgres, redis.
2. `docker-compose.dev.yml` overlays bind mounts and dev commands.
3. Backend boots, runs `alembic upgrade head`, then starts Uvicorn with reload in dev.
4. Frontend either serves built assets through Nginx or runs Vite dev server in the dev overlay.

### Manual/local path described in docs

1. `README.md` and `CLAUDE.md` describe a standalone `pgvector` container with different version/port/password assumptions.
2. Backend local commands use `uv`.
3. Frontend local commands use `npm`.

### Drift

- Compose says PostgreSQL 16 on `5432`.
- README/CLAUDE describe PostgreSQL 17 on `5433`.
- `.env.example` aligns more closely with compose than README/CLAUDE.

## Deployment Assumptions And Gaps

- The repo is optimized for local Compose-based development, not a clearly defined production deployment target.
- No orchestration manifests were found for ECS, Kubernetes, Nomad, or Terraform-managed infrastructure.
- No standalone worker container or scheduler leader-election mechanism exists.
- Frontend Nginx proxy assumes backend hostname `backend` on the same container network.

## Major Infra Risks

1. `frontend/vite.config.ts` targets `http://localhost:8000`, which breaks when the frontend dev server runs inside its own container.
2. The dev compose merge still retains the base frontend `3000:80` and Nginx healthcheck assumptions even though the dev service runs Vite on `5173`.
3. Secrets and origins in compose are development-grade and should not be mistaken for deployment-safe defaults.
4. README/CLAUDE/compose/env drift makes local boot and migration debugging harder than necessary.
5. Running scheduler jobs inside the API process makes horizontal scaling unsafe without duplication controls.
6. Backend container startup couples migration execution to app boot.
7. Scraping and browser-heavy automation may fail in the slim backend image if OS/browser dependencies are incomplete.
8. Redis TLS is not actually provisioned by repo contents; only a placeholder directory exists.
