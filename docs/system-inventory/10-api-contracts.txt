## Contract Rules

- Unless noted otherwise, the frontend calls these endpoints through axios with `withCredentials: true`.
- Unless noted otherwise, backend auth requirement is `get_current_user`, satisfied by auth cookies or bearer token.
- Query keys are page-local; there is no central registry.

## Auth

| Method / Path | Request Schema | Response Shape | Auth | Frontend Consumers / Query Keys | Backend Implementation | Notes |
|---|---|---|---|---|---|---|
| `POST /api/v1/auth/login` | Frontend `{ email, password }`; backend `LoginRequest(email, password)` | `TokenResponse { access_token, refresh_token, token_type }` | Public | `frontend/src/api/auth.ts`, `store/useAuthStore.ts`, `pages/Login.tsx` | `backend/app/auth/router.py -> login`, `backend/app/auth/service.py` | Sets auth cookies and returns tokens |
| `POST /api/v1/auth/register` | Frontend `{ email, password, display_name? }`; backend `UserCreate` | `UserResponse { id, email, display_name, is_active, created_at }` | Public | `frontend/src/api/auth.ts` | `backend/app/auth/router.py -> register`, `service.py` | Creates user only; does not establish client session by itself |
| `GET /api/v1/auth/me` | No body | `UserResponse` | Required | `store/useAuthStore.ts`, `AuthGuard`, `Profile`, `Settings` | `backend/app/auth/router.py -> me` | Current-user bootstrap endpoint |
| `POST /api/v1/auth/refresh` | Frontend sends `{}` or refresh cookie; backend accepts optional `RefreshRequest` | `TokenResponse` | Public by route, but requires refresh token or cookie | `api/client.ts` interceptor, `auth.ts`, auth store refresh path | `backend/app/auth/router.py -> refresh`, `service.py` | Critical refresh-on-401 contract |
| `POST /api/v1/auth/logout` | Empty body | `{ status: "ok" }` | Required | `store/useAuthStore.ts`, `AppShell` logout | `backend/app/auth/router.py -> logout` | Increments `token_version`, clears cookies |
| `POST /api/v1/auth/change-password` | Frontend `{ current_password, new_password }`; backend `ChangePasswordRequest` | `TokenResponse` | Required | `pages/Settings.tsx` | `backend/app/auth/router.py -> change_password_route`, `service.py` | Rotates tokens after password change |
| `DELETE /api/v1/auth/account` | No body | `204 No Content` | Required | `pages/Settings.tsx` | `backend/app/auth/router.py -> delete_account`, `admin/service.py` | Deletes current user after admin-style data clear |

## Known Contract Drift

1. `frontend/src/api/phase7a.ts` calls `GET /search-expansion`, but the backend only exposes `POST /api/v1/search-expansion/expand`.
2. `frontend/src/api/resume.ts` expects a narrow tailor result that does not match backend `ResumeTailorResponse`.
3. `frontend/src/api/salary.ts` expects salary field names and offer-evaluation fields that do not match backend salary schemas.
4. `frontend/src/api/scraper.ts` mismatches backend request/response shapes for scraper run, target detail, target trigger, release, and batch trigger paths, and expects extra career-page fields.
5. `frontend/src/api/admin.ts` and `frontend/src/api/phase7a.ts` use narrower or different shapes than backend dict/detail responses for admin and source-health flows.
6. `frontend/src/api/vault.ts` targets update routes whose backend router exists but whose service methods are missing.
