## DB Area Purpose Notes

| Area | Paths | Purpose |
|---|---|---|
| Engine/bootstrap | `D:/jobradar-v2/backend/app/database.py`, `D:/jobradar-v2/backend/app/dependencies.py` | Create DB engine/session and inject request-scoped sessions |
| Migration system | `D:/jobradar-v2/backend/app/migrations` | Schema evolution source of truth |
| ORM/entity definitions | Domain `models.py` files plus `resume/archetypes.py`, `scraping/dedup_feedback.py`, `auto_apply/form_learning.py` | Persistent table mapping |
| Query/data access | Domain `service.py` files and workers | Direct SQLAlchemy reads/writes; no repository package |
| Seed/bootstrap data | `D:/jobradar-v2/backend/seed_data.py` | Create/update demo user profile and resume seed state |
| Runtime references | `docker-compose.yml`, `.env.example`, `README.md`, `CLAUDE.md` | Runtime DB URL, container topology, local boot guidance |

## Source-Of-Truth Notes

- Database truth is distributed. To understand any table safely, all of the following matter:
  - model file
  - migration file(s)
  - service-layer query/write logic
  - runtime boot config
- No dedicated `db/` directory exists.
- Several features expose persistence through domain packages whose names do not obviously imply the underlying tables, especially `copilot`, `vault`, `resume/archetypes`, `auto_apply/form_learning`, and `scraping/dedup_feedback`.
