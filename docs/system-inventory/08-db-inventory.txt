## Transaction And Session Notes

- Each request gets an `AsyncSession` from `get_db()`.
- Services usually own commit boundaries themselves.
- Workers also perform direct commits.
- There is no explicit unit-of-work abstraction across multi-service workflows.
- Because `expire_on_commit=False`, post-commit objects remain usable without automatic reload.

## Database Risk Notes

1. `005_create_p2_tables.py` overlaps several branch-specific migrations and is the biggest clean-replay risk.
2. Postgres production behavior is richer than the SQLite test path, especially around indexes, JSON/array semantics, and pgvector/search features.
3. Integration secrets are stored in the DB, not just referenced externally.
4. Several important relationships remain soft links by company name/domain rather than foreign keys.
5. Search expansion persistence exists at the model layer but is only partially surfaced at the API/service layer.
