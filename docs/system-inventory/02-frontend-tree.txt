## Grouping By Concern

| Concern | Paths | Notes |
|---|---|---|
| Entry points | `D:/jobradar-v2/frontend/src/main.tsx`, `D:/jobradar-v2/frontend/src/App.tsx` | Runtime root, providers, route table |
| Pages/routes | `D:/jobradar-v2/frontend/src/pages` | Route containers are large and stateful |
| Shared layout/system | `D:/jobradar-v2/frontend/src/components/layout`, `D:/jobradar-v2/frontend/src/components/system` | Main shell grammar and reusable workspace composition |
| UI primitives | `D:/jobradar-v2/frontend/src/components/ui` | Safest reuse surface for refactors |
| Domain widgets | `D:/jobradar-v2/frontend/src/components/jobs`, `pipeline`, `scraper`, `analytics` | Mid-level feature composition |
| API layer | `D:/jobradar-v2/frontend/src/api` | Frontend/backend contract boundary |
| Local/global state | `D:/jobradar-v2/frontend/src/store`, `D:/jobradar-v2/frontend/src/hooks` | Zustand auth/UI state plus hook utilities |
| Tests | `D:/jobradar-v2/frontend/src/__tests__`, `components/__tests__`, `api/__tests__`, `hooks/__tests__` | Route, provider, API wrapper, and hook coverage |
| Config/design | `D:/jobradar-v2/frontend/system.md`, `vite.config.ts`, `vitest.config.ts`, `eslint.config.js` | Build, lint, test, and design-system rules |
