# Entrypoint: Plan → Implement → Review Workflow

> Canonical skill: `workflows/plan-implement-review`
> Canonical id: `workflows.plan-implement-review`

## What this workflow does

- Structures feature development into plan, implement, review, test, and wrap-up phases
- Assigns each phase to a canonical persona with explicit handoff contracts
- Provides quality gates between phases to catch issues early

## When to use

- Feature work touching 3+ files
- Tasks requiring planning before coding
- Any change that benefits from a review gate before commit

## When NOT to use

- Single-line fixes with obvious root cause (use quick-fix instead)
- Pure research tasks (use research-summarize-decide instead)
